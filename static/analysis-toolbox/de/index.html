GitAtbx

Ein modulares Nextflow-Pipeline-Framework für automatisierte Datenanalyse — mit eingebetteter Git-Synchronisierung der Ergebnisse.

Was ist GitAtbx?

GitAtbx bietet Infrastruktur für Datenpipelines, die mehrere Datensätze parallel verarbeiten, Ergebnisse automatisch via Git versionieren und den Fortschritt pro Teilnehmer verfolgen. Aufgebaut auf Nextflow für skalierbare, reproduzierbare Workflows.

Das Framework ist domänenagnostisch — Module folgen einfachen Ein-/Ausgabe-Konventionen (Parquet / FIF) und können beliebige Verarbeitungslogik implementieren. gitatbx init richtet ein neues Analyseprojekt ein: Verzeichnisstruktur, Konfigurationsdatei und optional ein GitHub-Remote für den Ergebnisordner.

Kernbefehle

BefehlFunktion
gitatbx init <dir>Neues Analyseprojekt anlegen
gitatbx run [pattern]Pipeline per Projektname-Suche starten
gitatbx serveErgebnis-HTML lokal im Browser anzeigen
gitatbx reinject <id>Korrigierten Output für einen Teilnehmer neu einspielen
gitatbx move <ziel>Module-Symlink verschieben
gitatbx config showAktuelle Konfiguration anzeigen

Installation

Installation per pip:

pip install GitAtbx

Voraussetzungen: Python 3.10+, Java Runtime + Nextflow, Git
