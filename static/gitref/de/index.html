GitRef

Ein git-basierter Literaturmanager — wie Zotero, aber die Bibliothek ist ein einfaches git-Repo.

Was ist GitRef?

GitRef verwaltet akademische Literatur mit Standard-BibTeX und git statt einer proprietären Datenbank. Jede Quelle, die per DOI, arXiv-ID oder ISBN hinzugefügt wird, wird in einer .bib-Datei gespeichert — PDFs landen komprimiert im Archiv. Die gesamte Bibliothek liegt in einem git-Repo unter eigener Kontrolle, synchronisiert sich über git push/pull und ist auf GitHub durchsuchbar.

Entwickelt für einfache Handhabung: gitref watch starten — PDFs werden entpackt. PDF in den Ordner legen — wird automatisch importiert. Betrachter schließen — Dateien werden automatisch neu gepackt.

Funktionen

  • DOI / arXiv / ISBN-Lookup — Identifier einfügen, Metadaten werden automatisch über Crossref, arXiv und Open Library abgerufen
  • PDF-Verwaltung — Automatischer Download, Umbenennung und Komprimierung in ein git-optimiertes Zip-Archiv
  • File Watcher — Entpackt PDFs beim Start, importiert neue Dateien automatisch, packt beim Schließen neu
  • BibTeX-nativ — Standardformat, kein Vendor-Lock-in, kompatibel mit LaTeX-Toolchains
  • Terminal-UI — Interaktiver Browser mit Suche, Detailansicht, Tags und Sammlungen
  • Browser-Extension — Ein-Klick-Import von jeder Paper-Seite (wie Zotero Connector)
  • Git-Sync — Automatisches Commit, Pull, Push über mehrere Geräte
  • RIS-Export — Kompatibel mit anderen Literaturverwaltungsprogrammen
  • Plattformübergreifend — Windows, Linux und macOS

Installation

Installation per pip:

pip install gitref

Voraussetzungen: Python 3.10+, Git (für Sync)
Lizenz: MIT
Status: Aktive Entwicklung

Schnellstart

# Neue Bibliothek initialisieren
gitref init

# Paper per DOI hinzufügen
gitref add "10.1038/s41586-024-07487-w"

# Paper per arXiv-ID hinzufügen
gitref add "2401.12345"

# Watcher starten (PDFs entpacken, Auto-Import, Auto-Repack)
gitref watch

# Interaktives TUI öffnen
gitref browse

# Mit Remote synchronisieren
gitref sync

# Vollständige Hilfe
gitref --help
